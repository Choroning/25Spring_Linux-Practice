## Q1. 줄, 단어, 문자 수 세기

**문제:** 텍스트 파일의 줄 수, 단어 수, 문자 수를 출력하는 Linux 명령어는?

**정답:** `wc`

**해설:** `wc`(word count) 명령어는 파일의 줄 수, 단어 수, 바이트(또는 문자) 수를 표시한다. 기본적으로 `wc filename`은 줄 수, 단어 수, 바이트 수의 세 값을 출력한다. 특정 수를 얻으려면 `-l`(줄), `-w`(단어), `-c`(바이트), 또는 `-m`(문자)을 사용한다.

---

## Q2. `wc -c`의 기능

**문제:** 다음 명령어는 무엇을 수행하는가?
```bash
$ wc -c /etc/passwd
```

**정답:** `/etc/passwd` 파일의 바이트 수를 출력한다.

**해설:** `wc -c` 옵션은 지정된 파일의 바이트 수를 세어 표시한다. 단일 바이트 인코딩(ASCII 등)을 사용하는 텍스트 파일의 경우 바이트 수와 문자 수가 같다. 다중 바이트 인코딩(UTF-8 등)의 경우 실제 문자 수를 얻으려면 `wc -m`을 사용해야 한다.

## Q3. 역순 정렬 옵션

**문제:** `sort` 명령어에서 역순으로 정렬하는 옵션은?

**정답:** `-r`

**해설:** `sort -r` 옵션은 기본 정렬 순서를 역전시켜, 내림차순으로 결과를 출력한다. 다른 옵션과 결합할 수 있으며, 예를 들어 `sort -rn`은 숫자를 내림차순으로 정렬하고, `sort -rk2`는 두 번째 필드를 역순으로 정렬한다.

## Q4. sort의 필드 구분자

**문제:** `sort` 명령어에서 필드 구분자를 `:`로 설정하는 옵션은?

**정답:** `-t:`

**해설:** `sort -t:` 옵션은 `:`를 필드 구분자로 지정한다. 이는 `/etc/passwd`와 같이 콜론으로 구분된 구조화된 데이터를 정렬할 때 필수적이다. 예를 들어, `sort -t: -k3 -n /etc/passwd`는 passwd 파일을 세 번째 필드(UID)를 기준으로 숫자순 정렬한다.

---

## Q5. 숫자순 정렬 옵션

**문제:** `sort` 명령어에서 숫자 필드를 숫자로 처리하는 옵션은?

**정답:** `-n`

**해설:** `sort -n` 옵션은 사전순 정렬 대신 숫자순 정렬을 수행한다. `-n` 없이는 문자열 "9"가 "10" 뒤에 정렬된다(사전순으로 '9' > '1'이므로). `-n`을 사용하면 숫자 값으로 비교하여 9가 올바르게 10 앞에 나타난다.

## Q6. `split`의 기본 파일 이름

**문제:** `split` 명령어가 파일을 생성할 때, 두 번째로 생성되는 파일의 이름은?

**정답:** `xab`

**해설:** 기본적으로 `split`은 `x` 접두사 뒤에 알파벳 접미사를 사용하여 출력 파일의 이름을 지정한다: `xaa`, `xab`, `xac` 등. 따라서 두 번째 파일은 `xab`이다. `split`의 두 번째 인자로 사용자 정의 접두사를 지정할 수 있으며, 예를 들어 `split -l 100 input.txt output_`는 `output_aa`, `output_ab` 등을 생성한다.

## Q7. `uniq`의 고유 줄만 출력

**문제:** `uniq` 명령어에서 중복되지 않은 줄만 출력하는 옵션은?

**정답:** `-u`

**해설:** `uniq -u` 옵션은 입력에서 반복되지 않는 줄만 출력한다. 연속으로 두 번 이상 나타나는 줄은 완전히 제외된다. `uniq`는 인접 중복만 감지하므로, 동일한 줄을 그룹화하려면 일반적으로 먼저 `sort`로 정렬해야 한다. `-d` 옵션은 반대로, 중복된 줄만 표시한다.

## Q8. `cut`의 필드 구분자

**문제:** `cut` 명령어에서 필드 구분자를 `:`로 설정하는 옵션은?

**정답:** `-d :`

**해설:** `cut -d :` 옵션(또는 `cut -d:`)은 `:`를 필드 구분자로 지정한다. 이것은 특정 필드를 추출하기 위해 `-f` 옵션과 함께 자주 사용된다. 예를 들어, `cut -d: -f1,3 /etc/passwd`는 passwd 파일에서 사용자 이름(필드 1)과 UID(필드 3)를 추출한다.

## Q9. 수평 병합

**문제:** `paste` 명령어에서 파일을 수평으로(직렬 모드) 병합하는 옵션은?

**정답:** `-s`

**해설:** `paste -s` 옵션은 입력을 직렬화하여, 각 파일의 모든 줄을 탭(또는 사용자 정의 구분자)으로 구분된 한 줄로 병합한다. `-s` 없이 `paste`는 파일을 줄 단위로(수직으로) 병합한다. 예를 들어, 파일에 세 줄이 있다면 `paste -s file`은 세 값을 모두 탭으로 구분된 한 줄로 출력한다.

## Q10. `dd`로 파일 생성

**문제:** 1 MB 파일을 생성하는 `dd` 명령어를 완성하시오.

**정답:** `dd if=/dev/zero bs=1M count=1 of=dd_test`

**해설:** `dd` 명령어는 블록 단위로 데이터를 복사한다. `if=/dev/zero`는 입력 소스(무한한 0 바이트 스트림)를 지정하고, `bs=1M`은 블록 크기를 1메가바이트로 설정하며, `count=1`은 정확히 1개 블록을 복사하고, `of=dd_test`는 출력 파일을 지정한다. 결과는 널 바이트로 채워진 1 MB 파일이다. `dd`는 디스크 이미징, 스왑 파일 생성, I/O 성능 벤치마킹에 흔히 사용된다.
