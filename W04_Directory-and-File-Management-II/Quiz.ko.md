## Q1. 디렉터리 복사

**문제:** `project/` 디렉터리 전체와 그 내용을 `/backup/`으로 올바르게 복사하는 명령어는?

| 선택지 | 명령어 |
|--------|--------|
| A | `cp project/ /backup/` |
| B | `cp -r project/ /backup/` |
| C | `cp -f project/ /backup/` |
| D | `cp -l project/ /backup/` |

**정답:** B. `cp -r project/ /backup/`

**해설:** `-r` (recursive) 옵션은 디렉터리와 모든 내용을 복사하는 데 필요하다. `-r` 없이 `cp`는 일반 파일만 복사하며 디렉터리가 주어지면 오류를 보고한다. `-f` 플래그는 강제 덮어쓰기만 하고 재귀 복사를 활성화하지 않는다. `-l` 플래그는 복사 대신 하드 링크를 생성한다.

## Q2. 하드 링크 vs. 심볼릭 링크

**문제:** 하드 링크에 대한 다음 설명 중 **틀린** 것은?

| 선택지 | 설명 |
|--------|------|
| A | 하드 링크는 원본 파일과 동일한 inode 번호를 공유한다 |
| B | 원본 파일을 삭제하면 하드 링크가 댕글링 링크가 된다 |
| C | 하드 링크는 서로 다른 파일시스템 간에 생성할 수 없다 |
| D | 하드 링크는 디렉터리에 대해 생성할 수 없다 |

**정답:** B. 원본 파일을 삭제하면 하드 링크가 댕글링 링크가 된다

**해설:** 하드 링크는 동일한 inode를 공유하므로, 최소 하나의 하드 링크가 남아 있는 한 파일의 데이터는 유지된다. "원본"을 삭제하면 단순히 하나의 디렉터리 항목이 제거되고 링크 수가 감소할 뿐이다. 데이터는 링크 수가 0이 될 때만 해제된다. 대상 파일이 삭제되었을 때 댕글링(끊어진) 링크가 되는 것은 심볼릭 링크이다.

## Q3. I/O 리다이렉션

**문제:** 출력 리다이렉션에서 `>`와 `>>`의 차이점은?

| 선택지 | 설명 |
|--------|------|
| A | `>`는 파일에 추가하고, `>>`는 파일을 덮어씀 |
| B | `>`는 파일을 덮어쓰고, `>>`는 파일에 추가함 |
| C | `>`와 `>>` 모두 파일을 덮어씀 |
| D | `>`는 stdout에 쓰고, `>>`는 stderr에 씀 |

**정답:** B. `>`는 파일을 덮어쓰고, `>>`는 파일에 추가함

**해설:** `>` 연산자는 출력을 파일로 리다이렉션하며, 파일이 존재하지 않으면 생성하고 존재하면 잘라내기(덮어쓰기)한다. `>>` 연산자도 출력을 리다이렉션하지만 파일의 끝에 추가하여 기존 내용을 보존한다. 예를 들어, `echo "line1" > log.txt`로 파일을 생성한 후, `echo "line2" >> log.txt`로 line1을 지우지 않고 두 번째 줄을 추가한다.

## Q4. grep과 정규 표현식 사용

**문제:** `/etc/passwd`에서 문자열 `root`로 시작하는 모든 줄을 표시하는 명령어는?

| 선택지 | 명령어 |
|--------|--------|
| A | `grep "root$" /etc/passwd` |
| B | `grep "^root" /etc/passwd` |
| C | `grep "*root" /etc/passwd` |
| D | `grep "root" /etc/passwd` |

**정답:** B. `grep "^root" /etc/passwd`

**해설:** `^` 앵커는 줄의 시작을 매칭하므로, `^root`는 "root"로 시작하는 줄을 매칭한다. 선택지 A는 줄의 끝을 매칭하는 `$`를 사용한다. 선택지 C는 정규식에서 "앞 문자의 0회 이상 반복"을 의미하는 `*`를 사용한다 (셸 글로빙의 와일드카드와 다름). 선택지 D는 줄의 시작뿐만 아니라 어디서든 "root"를 매칭한다.

## Q5. find로 파일 찾기

**문제:** `/var` 아래에서 50 MB보다 큰 모든 `.log` 파일을 찾는 명령어를 작성하시오.

**정답:** `find /var -name "*.log" -size +50M`

**해설:** `find` 명령어는 지정된 경로에서 재귀적으로 검색한다. `-name "*.log"` 옵션은 파일 이름 패턴으로 필터링하고, `-size +50M`은 50 메가바이트보다 큰 파일을 필터링한다. `+` 접두사는 "보다 큼"을 의미한다. `-type f`를 추가하여 일반 파일로만 검색을 제한할 수도 있다.

## Q6. 파이프 사용

**문제:** 다음 명령어는 무엇을 하는가?

```bash
cat /etc/passwd | sort | head -5
```

| 선택지 | 설명 |
|--------|------|
| A | `/etc/passwd`의 처음 5줄을 원래 순서로 표시 |
| B | `/etc/passwd`의 마지막 5줄을 알파벳 순으로 표시 |
| C | `/etc/passwd`의 처음 5줄을 알파벳 순으로 표시 |
| D | `/etc/passwd`를 정렬하고 처음 5줄을 파일에 저장 |

**정답:** C. `/etc/passwd`의 처음 5줄을 알파벳 순으로 표시

**해설:** 파이프(`|`)는 한 명령어의 stdout을 다음 명령어의 stdin으로 전달한다. 먼저, `cat /etc/passwd`가 파일 내용을 출력한다. 그런 다음 `sort`가 모든 줄을 알파벳 순으로 정렬한다. 마지막으로 `head -5`가 정렬된 출력에서 처음 5줄만 가져와 화면에 표시한다.

## Q7. tee 사용

**문제:** `ls -la`의 출력을 화면에서 보면서 동시에 `filelist.txt`에 저장하고 싶다. 어떤 명령어를 사용해야 하는가?

**정답:** `ls -la | tee filelist.txt`

**해설:** `tee` 명령어는 표준 입력에서 읽어서 표준 출력(화면)과 하나 이상의 파일에 동시에 기록한다. `tee` 없이는 출력을 보거나(`ls -la`) 저장하는 것(`ls -la > filelist.txt`) 중 하나만 선택해야 한다. 덮어쓰기 대신 추가하려면 `tee -a filelist.txt`를 사용한다.

## Q8. stderr 리다이렉션

**문제:** `find / -name "*.conf"`의 오류 메시지만 `errors.txt` 파일로 리다이렉션하고, 정상 출력은 화면에 표시하는 명령어는?

| 선택지 | 명령어 |
|--------|--------|
| A | `find / -name "*.conf" > errors.txt` |
| B | `find / -name "*.conf" 2> errors.txt` |
| C | `find / -name "*.conf" &> errors.txt` |
| D | `find / -name "*.conf" 1> errors.txt` |

**정답:** B. `find / -name "*.conf" 2> errors.txt`

**해설:** 파일 디스크립터 2는 stderr를 나타낸다. `2>`를 사용하면 오류 메시지("Permission denied" 등)만 지정된 파일로 리다이렉션하고, 정상 출력(stdout, 파일 디스크립터 1)은 계속 화면에 표시된다. 선택지 A는 stdout(stderr가 아님)을 리다이렉션한다. 선택지 C는 stdout과 stderr를 모두 리다이렉션한다. 선택지 D는 파일 디스크립터 1을 사용하여 stdout을 명시적으로 리다이렉션한다.
