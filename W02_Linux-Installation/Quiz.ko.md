## Q1. 터미널 프롬프트 기호

**문제:** 프롬프트 기호가 사용자의 권한 수준에 대해 무엇을 나타내는가?

| 선택지 | 설명 |
|--------|------|
| A | `$`는 root 사용자를, `#`은 일반 사용자를 나타냄 |
| B | `$`는 일반 사용자를, `#`은 root 사용자를 나타냄 |
| C | `$`와 `#` 모두 일반 사용자를 나타냄 |
| D | 프롬프트 기호는 사용자 권한과 관계 없음 |

**정답:** B. `$`는 일반 사용자를, `#`은 root 사용자를 나타냄

**해설:** Linux에서 셸 프롬프트는 일반(비root) 사용자에게는 `$`를, root(슈퍼유저)에게는 `#`을 표시한다. 이 시각적 단서는 사용자가 현재 권한 수준을 인식하도록 도와주며, root 권한으로 위험한 명령어를 실수로 실행하는 것을 방지하는 데 중요하다.

## Q2. 커널 버전 확인

**문제:** 현재 실행 중인 Linux 커널 버전을 올바르게 표시하는 명령어는?

| 선택지 | 명령어 |
|--------|--------|
| A | `kernel -v` |
| B | `uname -r` |
| C | `version --kernel` |
| D | `cat /etc/kernel` |

**정답:** B. `uname -r`

**해설:** `uname -r` 명령어는 커널 릴리스 버전(예: `5.15.0-56-generic`)을 출력한다. `-r` 플래그는 커널 릴리스 정보를 구체적으로 요청한다. `uname -a`를 사용하면 커널 이름, 호스트 이름, 버전, 아키텍처, 운영체제를 포함한 모든 시스템 정보가 표시된다.

## Q3. sudo의 목적

**문제:** 다음 중 `sudo`의 목적을 가장 잘 설명하는 것은?

| 선택지 | 설명 |
|--------|------|
| A | 현재 세션을 영구적으로 root로 전환 |
| B | 단일 명령어를 슈퍼유저 권한으로 실행 |
| C | 새로운 root 사용자 계정 생성 |
| D | root 계정 비활성화 |

**정답:** B. 단일 명령어를 슈퍼유저 권한으로 실행

**해설:** `sudo`(superuser do)는 허가된 사용자가 root 계정으로 전환하지 않고도 단일 명령어를 root로 실행할 수 있게 해준다. 명령어 실행이 완료되면 사용자는 일반 권한 수준으로 돌아간다. 이는 root로 직접 로그인하는 것보다 안전한데, 상승된 권한의 범위를 제한하고 감사 기록을 제공하기 때문이다.

## Q4. 시스템 종료

**문제:** 현재 시점으로부터 10분 후에 시스템 종료를 예약하는 명령어는?

| 선택지 | 명령어 |
|--------|--------|
| A | `sudo shutdown -h 10` |
| B | `sudo shutdown -h +10` |
| C | `sudo shutdown -r +10` |
| D | `sudo shutdown -c +10` |

**정답:** B. `sudo shutdown -h +10`

**해설:** `shutdown -h +10`은 현재 시간으로부터 10분 후에 정지(종료)를 예약한다. `-h` 플래그는 정지/전원 끄기를 의미하고, `+10`은 분 단위의 지연 시간을 지정한다. 선택지 A는 `+` 없이 `10`만 사용하여 다르게 해석된다. 선택지 C는 종료가 아닌 재부팅(`-r`)을 수행한다. 선택지 D는 예약된 종료를 취소하는 `-c`를 사용한다.

## Q5. VMware 스냅샷의 목적

**문제:** VMware 스냅샷 기능의 주요 목적은 무엇인가?

**정답:** 스냅샷은 특정 시점의 가상 머신 현재 상태를 저장하여, 실험 도중 문제가 발생했을 때 해당 상태로 VM을 복원할 수 있게 해준다.

**해설:** 스냅샷은 실습 환경에서 안전한 실험을 위해 매우 중요하다. 위험한 작업(시스템 파일 수정이나 테스트되지 않은 소프트웨어 설치 등)을 수행하기 전에 스냅샷을 생성하면, 알려진 정상 상태로 되돌릴 수 있다. 이는 전체 VM을 복제하는 Clone(복제)이나 VM을 일시적으로 중지하는 Suspend(일시 중지)와는 다르다.

## Q6. SSH 원격 접속

**문제:** `openssh-server`를 설치한 후, 매번 부팅할 때마다 SSH가 자동으로 시작되도록 하려면 어떤 명령어를 사용해야 하는가?

**정답:** `sudo systemctl enable ssh`

**해설:** `systemctl enable` 명령어는 systemd 시작 디렉터리에 필요한 심볼릭 링크를 생성하여 서비스가 부팅 시 자동으로 시작되도록 설정한다. 이는 현재 세션에서만 서비스를 즉시 시작하고 부팅 시 활성화하지 않는 `systemctl start ssh`와는 다르다. 활성화와 즉시 시작을 동시에 하려면 `sudo systemctl enable --now ssh`를 실행할 수 있다.

## Q7. Linux 특징

**문제:** 다음 중 Linux의 표준 특징이 아닌 것은?

| 선택지 | 특징 |
|--------|------|
| A | 다중 사용자 지원 |
| B | GPL 라이선스 하의 오픈소스 |
| C | 단일 작업 (한 번에 하나의 프로세스) |
| D | POSIX 호환성 |

**정답:** C. 단일 작업 (한 번에 하나의 프로세스)

**해설:** Linux는 다중 작업 운영체제로, 여러 프로세스를 동시에 실행할 수 있다. 다른 선택지는 모두 Linux의 올바른 특징이다: 여러 사용자의 동시 접속 지원(다중 사용자), GPL 라이선스 하에 소스 코드 자유 이용 가능(오픈소스), Unix 호환을 위한 완전한 POSIX 호환성.

## Q8. 명령줄 단축키

**문제:** 긴 명령어를 입력했지만 전체를 폐기하고 다시 시작해야 한다. 커서에서 줄의 시작까지 전부 삭제하는 키보드 단축키는?

| 선택지 | 단축키 |
|--------|--------|
| A | `Ctrl + K` |
| B | `Ctrl + U` |
| C | `Ctrl + L` |
| D | `Ctrl + C` |

**정답:** B. `Ctrl + U`

**해설:** `Ctrl + U`는 커서 위치에서 줄의 시작까지 모든 내용을 삭제한다. `Ctrl + K`는 커서에서 줄의 끝까지 삭제한다. `Ctrl + L`은 화면을 지우지만 현재 명령어는 유지된다. `Ctrl + C`는 현재 명령어/프로세스를 완전히 취소하고 새로운 프롬프트를 제공한다.
