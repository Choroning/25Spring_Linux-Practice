## Q1. Windows USB 드라이브 마운트

**문제:** Windows USB 드라이브를 마운트하는 올바른 명령어는?

| 선택지 | 명령어 |
|--------|---------|
| A | `mount -t ext3 /dev/sdb1 /mnt` |
| B | `mount -t xfs /dev/sdb1 /mnt` |
| C | `mount -t vfat /dev/sdb1 /mnt` |
| D | `mount -t iso9660 /dev/sdb1 /mnt` |

**정답:** C. `mount -t vfat /dev/sdb1 /mnt`

**해설:** Windows USB 드라이브는 일반적으로 FAT32 파일 시스템을 사용하며, Linux에서는 `vfat`에 해당한다. `ext3`와 `xfs`는 Linux 고유 파일 시스템이고, `iso9660`은 CD/DVD 미디어의 파일 시스템이다. Linux의 `vfat` 드라이버는 FAT16과 FAT32를 모두 지원하므로, Windows 포맷 USB 드라이브에 적합한 선택이다.

## Q2. 파티션 정보 확인

**문제:** 시스템에 설치된 모든 디스크의 파티션 정보를 확인하는 명령어는?

| 선택지 | 명령어 |
|--------|---------|
| A | `fdisk -a` |
| B | `fdisk -l` |
| C | `fdisk -v` |
| D | `fdisk -d` |

**정답:** B. `fdisk -l`

**해설:** `fdisk -l` 명령어는 시스템의 모든 디스크에 대한 파티션 테이블을 나열한다. 장치 이름, 부팅 플래그, 시작/끝 섹터, 크기, 파티션 유형 등의 정보를 표시한다. 나머지 옵션은 유효하지 않거나 다른 목적으로 사용된다(`-v`는 버전 정보를 표시).

## Q3. ext3 파일 시스템 생성

**문제:** ext3 파일 시스템을 생성하는 명령어로 **올바르지 않은** 것은?

| 선택지 | 명령어 |
|--------|---------|
| A | `mkfs -t ext3 /dev/sdb1` |
| B | `mke2fs -t ext3 /dev/sdb1` |
| C | `mkfs.ext3 /dev/sdb1` |
| D | `mke2fs.ext3 /dev/sdb1` |

**정답:** D. `mke2fs.ext3 /dev/sdb1`

**해설:** 선택지 A, B, C는 모두 ext3 파일 시스템을 생성하는 유효한 방법이다. `mkfs -t ext3`는 `-t` 플래그로 유형을 지정하고, `mke2fs -t ext3`는 ext2/3/4 전용 도구를 사용하며, `mkfs.ext3`는 약식 명령어이다. 그러나 `mke2fs.ext3`는 시스템에 존재하지 않는 유효하지 않은 명령어이다.

## Q4. 디스크 기반이 아닌 파일 시스템

**문제:** 다음 중 디스크 기반 파일 시스템이 **아닌** 것은?

| 선택지 | 파일 시스템 |
|--------|-------------|
| A | proc |
| B | ext |
| C | ext4 |
| D | xfs |

**정답:** A. proc

**해설:** `proc` 파일 시스템(procfs)은 커널 데이터 구조에 대한 인터페이스를 제공하는 가상/의사 파일 시스템이다. 디스크에 존재하지 않고 메모리에서만 존재하며, `/proc`을 통해 프로세스와 시스템 상태에 대한 정보를 제공한다. 반면에 `ext`, `ext4`, `xfs`는 모두 물리적 저장 장치에 데이터를 저장하는 디스크 기반 파일 시스템이다.

## Q5. 디스크 설정 절차

**문제:** 새로 설치한 디스크를 설정할 때 올바른 작업 순서는?

| 선택지 | 순서 |
|--------|-------|
| A | 파티션 생성 -> 디스크 마운트 -> 디스크 포맷 |
| B | 파티션 생성 -> 디스크 포맷 -> 디스크 마운트 |
| C | 디스크 마운트 -> 파티션 생성 -> 디스크 포맷 |
| D | 디스크 포맷 -> 파티션 생성 -> 디스크 마운트 |

**정답:** B. 파티션 생성 -> 디스크 포맷 -> 디스크 마운트

**해설:** 새 디스크를 설정하는 올바른 절차는 다음과 같다: (1) `fdisk` 또는 `parted`를 사용하여 파티션 생성, (2) `mkfs`를 사용하여 파티션에 파일 시스템으로 포맷, (3) `mount`를 사용하여 포맷된 파티션을 디렉터리에 마운트. 파티션 없이 포맷할 수 없으며, 마운트에는 포맷된 파일 시스템이 필요하다.

## Q6. 부팅 시 마운트 설정

**문제:** 부팅 시 파일 시스템을 마운트하기 위한 정보를 저장하는 파일은?

| 선택지 | 파일 |
|--------|------|
| A | `/etc/mtab` |
| B | `/etc/mount` |
| C | `/etc/fstab` |
| D | `/etc/ftab` |

**정답:** C. `/etc/fstab`

**해설:** `/etc/fstab`(파일 시스템 테이블) 파일은 부팅 시 마운트할 파일 시스템에 대한 정적 정보를 포함한다. 각 항목은 장치, 마운트 포인트, 파일 시스템 유형, 마운트 옵션, dump 빈도, fsck 패스 번호를 지정한다. `/etc/mtab`는 현재 마운트된 파일 시스템을 추적하지만 부팅 시 설정에는 사용되지 않는다.

## Q7. 마운트 포인트

**문제:** 디렉터리 계층 구조에서 파일 시스템이 연결되는 디렉터리를 무엇이라 하는가?

**정답:** 마운트 포인트(Mount point)

**해설:** 마운트 포인트는 기존 파일 시스템 계층 구조에서 추가 파일 시스템이 부착(마운트)되는 디렉터리이다. 파일 시스템이 디렉터리에 마운트되면, 해당 파일 시스템의 내용이 그 디렉터리 경로를 통해 접근 가능해진다. 일반적인 마운트 포인트로는 `/`, `/home`, `/mnt`, `/media` 등이 있다.

## Q8. CD-ROM 마운트

**문제:** CD-ROM을 `/cdrom` 디렉터리에 마운트하는 명령어는?

**정답:** `sudo mount -t iso9660 /dev/cdrom /cdrom`

**해설:** CD-ROM은 ISO 9660 파일 시스템 표준을 사용한다. `-t iso9660` 옵션은 이 파일 시스템 유형을 지정한다. `/dev/cdrom`은 CD-ROM 드라이브를 나타내는 장치 파일이고, `/cdrom`은 마운트 포인트 디렉터리이다. 마운트는 일반적으로 root 권한이 필요하므로 `sudo`를 사용한다.

## Q9. 파티션에 파일 시스템 생성

**문제:** 파티션에 파일 시스템을 생성하는 데 사용하는 명령어는?

**정답:** `sudo mkfs -t <filesystem-type> /dev/sdXN`

**해설:** `mkfs`(make file system) 명령어는 지정된 파티션에 파일 시스템을 생성한다. `-t` 옵션은 파일 시스템 유형(예: ext4, xfs, vfat)을 지정한다. 장치 경로 `/dev/sdXN`은 특정 파티션을 가리키며, `X`는 디스크 문자이고 `N`은 파티션 번호이다(예: `/dev/sdb1`).

## Q10. LVM의 목적

**문제:** LVM(Logical Volume Manager)과 같은 기능이 필요한 이유를 설명하라.

**정답:** LVM은 유연한 디스크 관리, 볼륨의 동적 확장/축소, 스냅샷 기능, 디스크 교체/복구 기능을 제공한다.

**해설:** LVM은 물리적 저장소 계층을 추상화하여 관리자가 언마운트 없이 볼륨 크기를 조정하고, 백업용 스냅샷을 생성하며, 여러 물리 디스크에 걸쳐 볼륨을 확장하고, 무중단으로 디스크 간 데이터를 마이그레이션할 수 있게 한다. LVM 없이는 파티션 크기 조정에 언마운트가 필요하고 잠재적으로 데이터를 잃을 수 있어 저장소 관리가 경직되고 오류가 발생하기 쉽다.
