## Q1. 스크립트의 정의

**문제:** 인터프리터라 불리는 다른 프로그램에 의해 실행되는 프로그램을 무엇이라 하는가? 인터프리터 프로그램의 예로는 Python, Bash, sh 등이 있다.

**정답:** 스크립트(Script)

**해설:** 스크립트는 사전에 기계어로 컴파일되는 것이 아니라, 런타임에 인터프리터에 의해 해석되고 실행되는 스크립트 언어로 작성된 프로그램이다. 셸 스크립트는 bash나 sh 같은 셸 프로그램에 의해 해석되며, Python 스크립트는 Python 인터프리터에 의해 해석된다.

## Q2. 매직 넘버 (셔뱅)

**문제:** 셸 스크립트 파일의 맨 처음에 오는 다음 줄을 무엇이라 부르는가?
```bash
#!/bin/bash
```

**정답:** 매직 넘버(Magic number) / 셔뱅(Shebang)

**해설:** 스크립트 시작 부분의 `#!` 시퀀스를 셔뱅(또는 매직 넘버)이라 한다. 이것은 스크립트를 실행할 때 어떤 인터프리터를 사용할지 커널에 알려준다. `#!` 뒤의 경로는 인터프리터 바이너리의 절대 경로를 지정한다. 예를 들어, `#!/bin/bash`는 `/bin/bash`에 위치한 Bash 셸을 사용하여 스크립트를 실행해야 함을 의미한다.

## Q3. 셸 스크립트의 주석

**문제:** 셸 스크립트에서 주석은 (1)로 시작하며, (1)부터 줄 끝까지 모두 주석으로 처리된다. (1)은 무엇인가?

**정답:** `#`

**해설:** bash 및 기타 셸 스크립트 언어에서 `#` 문자는 주석의 시작을 표시한다. `#`부터 해당 줄의 끝까지 모든 내용은 인터프리터에 의해 무시된다. 유일한 예외는 첫 번째 줄의 셔뱅(`#!`)으로, 이것은 셸이 아닌 커널에 의해 해석된다.

## Q4. 기본값을 포함한 변수 치환

**문제:** 다음 셸 스크립트의 출력은 무엇인가?
```bash
#!/bin/bash
a=red
echo ${a:-blue} ${b:=blue}
```

**정답:** `red blue`

**해설:** `${a:-blue}`는 `a`가 설정되어 있고 비어있지 않으면 `a`의 값을 반환하고, 그렇지 않으면 `blue`를 반환한다. `a=red`이므로 결과는 `red`이다. `${b:=blue}`는 `b`가 설정되지 않았거나 비어 있으면 `b`에 `blue`를 할당한 후 그 값을 반환한다. `b`가 설정되지 않았으므로 `blue`가 할당되고 `blue`가 출력된다.

## Q5. 문자열 패턴 제거

**문제:** 다음 셸 스크립트의 출력은 무엇인가?
```bash
#!/bin/bash
path="/usr/local/bin/tmskim/bin/file"
echo ${path##*/}
```

**정답:** `file`

**해설:** `${path##*/}`는 변수에서 `*/` 패턴과 일치하는 가장 긴 접두사를 제거한다. `##` 연산자는 탐욕적(가장 긴 일치) 접두사 제거를 수행한다. `*/` 패턴은 마지막 `/`까지의 모든 것과 일치하므로, `file`만 남게 된다. 이것은 전체 경로에서 파일 이름을 추출하는 일반적인 관용구로, `basename`과 동일한 역할을 한다.

## Q6. 명령줄 인자 개수 세기

**문제:** 다음 스크립트를 `./a.sh 1 2 3 4 5 6`으로 실행하면 무엇이 출력되는가?
```bash
#!/bin/bash
echo $#
```

**정답:** `6`

**해설:** 특수 변수 `$#`는 스크립트에 전달된 위치 매개변수(명령줄 인자)의 수를 저장한다. 스크립트가 6개의 인자(`1 2 3 4 5 6`)로 호출되었으므로, `$#`는 `6`으로 평가된다.

## Q7. 모든 인자 합산

**문제:** 다음 스크립트는 `./a.sh 1 2 3 4 5 6`으로 실행할 때 모든 명령줄 인자를 합산한다. (1)에 들어갈 것은?
```bash
#!/bin/bash
sum=0
for i in (1)
do
  (( sum = sum + i ))
done
echo $sum
```

**정답:** `"$@"`

**해설:** `"$@"`는 모든 위치 매개변수를 각각의 개별 토큰으로 유지하면서 별도의 단어로 확장한다. `for` 반복문에서 이것은 각 인자를 하나씩 순회한다. `"$*"`는 모든 인자를 하나의 문자열로 연결한다. `$@` 주위의 따옴표는 공백이 포함된 인자를 올바르게 처리하는 데 중요하다.

## Q8. `-x`를 사용한 디버그 모드

**문제:** `#!/bin/bash -x`에서 `-x`의 역할은?
```bash
#!/bin/bash -x
i=0
while (( i<10 ))
do
  echo $i
  ((i++))
done
```

**정답:** 각 줄을 실행하기 전에 터미널에 출력한다 (디버그/추적 모드).

**해설:** `-x` 옵션은 bash에서 xtrace(실행 추적) 모드를 활성화한다. 활성화되면, bash는 각 명령을 실행하기 전에 `+` 접두사를 붙여 stderr에 출력한다. 이는 모든 변수 확장이 적용된 실제 실행 명령을 보여주므로, 스크립트 디버깅에 매우 유용하다.

## Q9. 파일 존재 테스트

**문제:** 다음 스크립트는 명령줄 인자로 주어진 파일이 존재하는지 확인한다. (1)에 들어갈 것은?
```bash
#!/bin/bash
if (1)
then
  echo "$1 exist"
else
  echo "$1 does not exist"
fi
```
사용법: `./a.sh /bin/lls`

**정답:** `[ -e "$1" ]` (또는 `[[ -e "$1" ]]`)

**해설:** `-e` 테스트 연산자는 파일 유형(일반 파일, 디렉터리, 심볼릭 링크 등)에 관계없이 파일이 존재하는지 확인한다. `$1`은 첫 번째 명령줄 인자를 참조한다. `"$1"`처럼 따옴표를 사용하면 공백이 포함된 파일 이름을 처리할 수 있다. `[ ]`(test)과 `[[ ]]`(bash 확장 test) 문법 모두 유효하다.

## Q10. 중괄호 확장을 이용한 구구단

**문제:** 다음 스크립트는 주어진 인자에 대한 구구단을 출력한다. (1)에 들어갈 것은?
```bash
#!/bin/bash
for i in (1)
do
  printf "%2d x %2d = %2d " $1 $i $(($1 * $i))
  printf "\n"
done
```
사용법: `./a.sh 5`는 `5 x 1 = 5`부터 `5 x 9 = 45`까지 출력한다.

**정답:** `{1..9}`

**해설:** 중괄호 확장 `{1..9}`는 1부터 9까지의 정수 시퀀스를 생성한다. `for` 반복문에서 `i`는 1부터 9까지 각 값을 순회하며, 각 반복에서 `$1 * $i`를 계산한다. 이것은 bash 고유 기능이며, POSIX sh에서는 `seq 1 9` 또는 `$(seq 1 9)`를 대신 사용한다.

## Q11. 디렉터리 내 스크립트 파일 수 세기

**문제:** 디렉터리를 명령줄 인자로 받아 해당 디렉터리 내 스크립트 파일(`.sh` 파일)의 수를 출력하는 스크립트 `a.sh`를 작성하시오.
```
$ ./a.sh /tmp
6
```

**정답:**
```bash
#!/bin/bash
dir="$1"
if [ ! -d "$dir" ]; then
    echo "Usage: $0 <existing-directory>"
    exit 1
fi
count=$(find "$dir" -type f -name '*.sh' | wc -l)
echo $count
```

**해설:** 스크립트는 `[ ! -d "$dir" ]`를 사용하여 인자가 존재하는 디렉터리인지 검증한다. 그런 다음 `find`를 사용하여 지정된 디렉터리에서 `*.sh` 패턴에 일치하는 일반 파일(`-type f`)을 재귀적으로 검색한다. 출력은 `wc -l`로 파이프하여 일치하는 줄(파일) 수를 세고, 결과를 출력한다.
