## 1. Linux란 무엇인가?

Linux는 Unix 운영체제에 기반한 **무료 오픈소스 운영체제** 이다.

### 1.1 주요 특징

- **오픈소스:** 누구나 소스 코드를 자유롭게 열람, 수정, 배포할 수 있음
- **Unix 호환:** Unix 표준과 완벽한 호환성 유지
- **다중 사용자 및 다중 작업:** 여러 사용자와 프로세스를 동시에 지원
- **안정성 및 보안:** 견고한 보안과 시스템 안정성으로 유명
- **유연성:** 서버에서 스마트폰까지 다양한 장치에서 실행 가능

### 1.2 왜 Linux를 배워야 하는가?

- 전 세계 스마트폰의 **80%** 이상이 Android(Linux 기반)를 실행
- 클라우드 서버의 **95%** 이상이 Linux를 실행
- 세계 상위 500대 슈퍼컴퓨터의 **100%** 가 Linux를 실행
- 세계 상위 25개 웹사이트 대부분이 Linux에서 운영
- 5G 인프라, IoT 장치, 드론, 자율주행차에 필수적

> **핵심 포인트:** Linux는 단순한 서버 운영체제가 아니라 현대 컴퓨팅 인프라의 기반이다.

## 2. Linux의 역사

### 2.1 Linus Torvalds

- **출생:** 1969년 12월 28일, 핀란드 헬싱키
- **학력:** 헬싱키 대학교
- **1991년:** Minix(교육용 운영체제)를 참고하여 Linux를 개발
- **1991년 8월 26일:** Linux의 공식 생일

```
"Hello everybody out there using minix -
I'm doing a (free) operating system (just a hobby, won't be big
and professional like gnu) for 386(486) AT clones."
-- Linus Torvalds (1991)
```

### 2.2 Richard Stallman

- 1953년 뉴욕 맨해튼 출생
- 16세에 IBM 인턴으로 컴퓨터 작업 시작
- 1974년 MIT AI 연구소에서 **Emacs** 편집기 개발
- 소프트웨어 상업화에 반대하고 협력 개발 문화를 주장
- 1985년 **자유 소프트웨어 재단(FSF)** 설립
- **GNU 일반 공중 사용 허가서(GPL)** 작성
- **Copyleft** 운동을 시작

### 2.3 연표

| 연도 | 사건 |
|------|------|
| 1969 | AT&T Bell 연구소에서 Unix 개발 |
| 1983 | Richard Stallman이 GNU 프로젝트 발표 |
| 1985 | 자유 소프트웨어 재단(FSF) 설립 |
| 1991 | Linus Torvalds가 Linux 커널 0.01 공개 |
| 1993 | Debian 및 Slackware 배포판 출시 |
| 1994 | Red Hat Linux 출시 |
| 2004 | Ubuntu 출시 (Debian 기반) |
| 2025 | 최신 커널 버전: 6.13.5 |

## 3. GNU 프로젝트와 자유 소프트웨어

### 3.1 GNU 프로젝트란?

GNU는 **"GNU's Not Unix"** 의 약자로, 재귀적 약어이다. Unix와 호환되는 자유 소프트웨어를 개발하기 위한 프로젝트이다.

### 3.2 자유 소프트웨어의 네 가지 자유

| 자유 | 설명 |
|------|------|
| 자유 0 | 어떤 목적으로든 프로그램을 실행할 자유 |
| 자유 1 | 프로그램이 어떻게 작동하는지 연구하고 수정할 자유 (소스 코드 접근 필요) |
| 자유 2 | 다른 사람을 돕기 위해 복사본을 재배포할 자유 |
| 자유 3 | 수정된 버전을 배포하여 커뮤니티가 혜택을 받을 수 있게 할 자유 |

### 3.3 GPL (GNU 일반 공중 사용 허가서)

- 파생 저작물이 자유 소프트웨어로 유지되도록 보장하는 **Copyleft** 라이선스
- 버전: GPLv1, GPLv2, GPLv3
- Linux 커널은 **GPLv2** 라이선스를 사용

> **핵심 포인트:** "자유 소프트웨어"는 가격이 아닌 자유를 의미한다. "무료 맥주(free beer)"가 아닌 "언론의 자유(free speech)"로 이해해야 한다.

## 4. Linux 배포판

Linux 배포판은 **Linux 커널** 과 **응용 프로그램** 을 함께 묶어 구성한 것이다.

### 4.1 주요 배포판 계열

```
Linux Kernel (1991)
    |
    +-- Debian (1993)
    |       +-- Ubuntu (2004) --> Linux Mint (2006)
    |
    +-- Slackware (1993)
    |       +-- S.u.S.E (1994) --> SuSE (1998) --> openSUSE (2006)
    |
    +-- Red Hat (1994)
            +-- RHEL 2.1 (2003) --> RHEL 9 (2022)
            +-- Fedora Core (2003) --> Fedora 36 (2022)
            +-- CentOS (2003) --> CentOS Stream 9 (2021)
```

### 4.2 인기 배포판

| 배포판 | 기반 | 용도 | 웹사이트 |
|--------|------|------|----------|
| **Ubuntu** | Debian | 데스크톱, 서버 | https://ubuntu.com |
| **CentOS/RHEL** | Red Hat | 엔터프라이즈 서버 | https://centos.org |
| **Fedora** | Red Hat | 개발 | https://getfedora.org |
| **Arch Linux** | 독립 | 고급 사용자 | https://archlinux.org |

> **핵심 포인트:** Ubuntu는 초보자에게 가장 인기 있는 배포판이며, 본 강의에서 사용한다.

## 5. Linux 아키텍처

### 5.1 시스템 구조

```
+-----------------------------------+
|       응용 프로그램               |
|  (텍스트 편집기, 컴파일러 등)     |
+-----------------------------------+
|            Shell                   |
|   (명령어 해석기: bash)           |
+-----------------------------------+
|          Linux 커널               |
|  (프로세스, 파일시스템, 메모리,   |
|   장치, 네트워크 관리)           |
+-----------------------------------+
|           하드웨어                |
|  (CPU, 메모리, 디스크, NIC 등)   |
+-----------------------------------+
```

### 5.2 구성 요소의 역할

- **커널:** 운영체제의 핵심으로, 하드웨어 자원, 프로세스, 메모리, 파일시스템을 관리
- **Shell:** 사용자와 커널 사이의 인터페이스로, 명령어를 해석하고 실행
- **응용 프로그램:** 사용자가 상호작용하는 프로그램 (편집기, 컴파일러, 웹 서버)

## 7. 강의 개요

### 7.1 강의 정보

- **교과목 코드:** DCCS209 리눅스 실습
- **교재:** 이종원 저, "IT CookBook: 우분투 리눅스" (3판), 한빛아카데미, 2022
- **참고서:** 우재남 저, "IT CookBook: 초보자를 위한 리눅스 실습," 한빛아카데미, 2022
- **실습 환경:** VMware Workstation 위에서 Ubuntu Linux 사용

### 7.2 가상 머신 구성

**가상 머신(VM)** 은 호스트 운영체제 위에 가상 시스템을 생성하여 게스트 운영체제를 설치하고 실행할 수 있게 해준다.

| VM 소프트웨어 | 호스트 OS | 게스트 OS |
|--------------|-----------|-----------|
| VMware | Windows, Linux, macOS | Windows, Linux, Solaris, macOS |
| VirtualBox | Windows, Linux, macOS, Solaris | Windows, Linux, Solaris, macOS, OpenBSD |
| Hyper-V | Windows | Windows, 일부 Linux, Solaris |

**VMware 유용한 기능:**
- **Clone(복제):** 테스트를 위해 전체 VM을 복제
- **Suspend(일시 중지):** VM 상태를 저장하고 복원
- **Snapshot(스냅샷):** 나중에 되돌릴 수 있는 체크포인트 저장

### 7.3 평가 기준

| 구성 요소 | 비중 |
|-----------|------|
| 출석 | 10% |
| 실습 과제 (~10회) | 30% |
| 중간고사 | 30% |
