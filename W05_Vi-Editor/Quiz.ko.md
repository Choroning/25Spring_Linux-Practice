## Q1. Vi 동작 모드

**문제:** 명령 모드에서 입력 모드로 전환할 때, 커서 **앞에** 텍스트를 삽입하는 키는?

| 선택지 | 키 |
|--------|-----|
| A | `a` |
| B | `i` |
| C | `o` |
| D | `A` |

**정답:** B. `i`

**해설:** `i`는 입력 모드에 진입하며 현재 문자 앞에 커서를 위치시켜 삽입한다. `a`는 커서 뒤에 추가한다. `o`는 현재 줄 아래에 새 줄을 연다. `A`는 현재 줄의 끝에 추가한다. 네 가지 모두 입력 모드에 진입하지만, 텍스트 입력이 시작되는 위치가 다르다.

## Q2. Vi 저장 및 종료

**문제:** vi에서 파일을 변경했지만 변경 사항을 **저장하지 않고** 종료하고 싶다. 올바른 명령어는?

| 선택지 | 명령어 |
|--------|--------|
| A | `:wq` |
| B | `:q` |
| C | `:q!` |
| D | `ZZ` |

**정답:** C. `:q!`

**해설:** `:q!`는 수정 사항을 저장하지 않고 vi를 강제로 종료한다. `:q`는 변경 사항이 없을 때만 작동한다. `:wq`는 변경 사항을 저장한 후 종료한다. `ZZ`(명령 모드에서)는 저장 후 종료하며 `:wq`와 동일하다. `:q!`의 `!`는 저장하지 않은 변경 사항이 있을 때 종료를 방지하는 vi의 안전 장치를 무시한다.

## Q3. 줄 삭제

**문제:** 명령 모드에서 현재 줄과 그 아래 4줄(총 5줄)을 삭제하고 싶다. 어떤 명령어를 입력해야 하는가?

| 선택지 | 명령어 |
|--------|--------|
| A | `d5` |
| B | `5dd` |
| C | `5dw` |
| D | `d5d` |

**정답:** B. `5dd`

**해설:** vi에서 명령어 앞의 숫자는 해당 명령어를 그 횟수만큼 반복한다. `5dd`는 현재 줄부터 시작하여 5줄을 삭제한다. `dw`는 줄이 아닌 단어를 삭제하므로 `5dw`는 5개의 단어를 삭제한다. `d5` 형식은 불완전하여 이동 명령을 기다린다. `d5d`는 유효한 vi 구문이 아니다.

## Q4. 검색 및 바꾸기

**문제:** 전체 파일에서 `foo`의 **모든** 항목을 `bar`로 바꾸면서, 각 교체 전에 확인을 요청하는 명령어는?

| 선택지 | 명령어 |
|--------|--------|
| A | `:s/foo/bar/g` |
| B | `:%s/foo/bar/g` |
| C | `:%s/foo/bar/gc` |
| D | `:%s/foo/bar/` |

**정답:** C. `:%s/foo/bar/gc`

**해설:** `%`는 전체 파일을 의미하고, `s`는 치환 명령어이며, `g`는 각 줄에서 첫 번째뿐만 아니라 모든 항목을 바꾸고, `c`는 각 교체 전에 확인 프롬프트를 추가한다. `%` 없이는 현재 줄에만 치환이 적용된다. `g` 없이는 각 줄의 첫 번째 항목만 교체된다. `c` 없이는 교체가 자동으로 수행된다.

## Q5. 커서 탐색

**문제:** 큰 파일의 처음에 있으며 50번째 줄로 이동하고 싶다. 다음 중 유효하지 **않은** 방법은?

| 선택지 | 명령어 |
|--------|--------|
| A | `50G` (명령 모드에서) |
| B | `:50` (마지막 줄 모드에서) |
| C | `50gg` (명령 모드에서) |
| D | `G50` (명령 모드에서) |

**정답:** D. `G50` (명령 모드에서)

**해설:** `50G`는 줄 번호 접두사와 `G` (줄 이동) 명령어를 사용하여 50번째 줄로 이동한다. `:50`은 마지막 줄 모드에서 동일한 결과를 달성한다. `50gg`도 `gg`에 숫자를 지정하여 해당 줄로 이동하므로 유효하다. 그러나 `G50`은 유효하지 않다 — vi에서 숫자는 명령어 앞에 와야 하며, 뒤에 올 수 없다. `G`만 사용하면 파일의 마지막 줄로 이동한다.

## Q6. Vi에서의 복사 및 붙여넣기

**문제:** 현재 줄부터 시작하여 3줄을 복사하고 현재 커서 위치 아래에 붙여넣는 방법을 설명하시오.

**정답:** `3yy`를 입력하여 3줄을 복사(yank)한 후, 커서를 원하는 위치로 이동하고 `p`를 눌러 현재 줄 아래에 붙여넣는다.

**해설:** `yy`(또는 `Y`)는 현재 줄을 기본 버퍼에 복사한다. 앞에 숫자를 붙이면(`3yy`) 해당 줄 수만큼 복사한다. `p` 명령어는 버퍼 내용을 현재 줄 아래에 붙여넣고, `P`는 위에 붙여넣는다. 복사된 텍스트는 버퍼에 남아 있어 여러 번 붙여넣기가 가능하다.

## Q7. Vi 환경 설정

**문제:** 매번 vim을 열 때마다 `:set nu`를 입력하지 않고도 줄 번호가 항상 표시되도록 하려면 어떻게 해야 하는가?

**정답:** `~/.vimrc` 설정 파일에 `set number`를 추가한다.

**해설:** `~/.vimrc` 파일은 vim이 시작될 때마다 읽히는 사용자별 설정 파일이다. 앞의 콜론 없이 모든 `:set` 명령어를 여기에 넣을 수 있다. 예를 들어, `set number`를 추가하면 줄 번호가 영구적으로 활성화되고, `set autoindent`는 자동 들여쓰기를 활성화하며, `set tabstop=4`는 탭 너비를 4칸으로 설정한다.

## Q8. Vi에서 셸 명령어 실행

**문제:** vi에서 파일을 편집 중인데 편집기를 종료하지 않고 현재 디렉터리 목록을 확인해야 한다. 이를 가능하게 하는 명령어는?

| 선택지 | 명령어 |
|--------|--------|
| A | `:ls` |
| B | `:!ls` |
| C | `:shell ls` |
| D | `:run ls` |

**정답:** B. `:!ls`

**해설:** vi의 `:!command` 구문은 편집기를 종료하지 않고 외부 셸 명령어를 실행한다. 명령어가 실행되고 출력이 표시된 후, Enter를 누르면 vi로 돌아간다. 이는 코드 컴파일(`:!gcc program.c`), 파일 목록 확인, 또는 편집 도중 셸 명령어를 실행하는 데 유용하다. vim에서 `:ls`는 디스크의 파일이 아닌 열린 버퍼를 나열한다.
