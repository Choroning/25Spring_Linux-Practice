## Q1. 중복 UID/GID를 지원하는 명령어

**문제:** 사용자 계정 그룹과 관련된 명령어 중 중복 UID 또는 GID 설정을 **허용하지 않는** 명령어는?

| 선택지 | 명령어 |
|--------|--------|
| A | `usermod` |
| B | `useradd` |
| C | `userdel` |
| D | `groupadd` |

**정답:** C. `userdel`

**해설:** `userdel` 명령어는 사용자 계정을 삭제하는 데 사용되며, UID나 GID 값 설정과는 전혀 관련이 없다. `usermod`, `useradd`, `groupadd`와 같은 명령어는 비고유(중복) UID 또는 GID 할당을 허용하는 `-o` 옵션을 지원한다. `userdel`은 계정 제거만 수행하므로, 중복 UID/GID 개념이 적용되지 않는다.

## Q2. 특정 UID로 사용자 생성

**문제:** `adduser` 명령어를 사용하여 UID 1010으로 사용자 계정 `han01`을 생성하는 방법은?

| 선택지 | 명령어 |
|--------|--------|
| A | `adduser 1010 han01` |
| B | `adduser -u 1010 han01` |
| C | `adduser -uid 1010 han01` |
| D | `adduser --uid 1010 han01` |

**정답:** D. `adduser --uid 1010 han01`

**해설:** Debian/Ubuntu의 `adduser` 명령어는 사용자 지정 UID를 지정하기 위해 긴 옵션 `--uid`를 사용한다. `adduser`(Debian 전용, 상위 수준 래퍼)는 `useradd`(저수준, 표준)와 다르다는 점에 주의해야 한다. `useradd`에서는 짧은 옵션 `-u`가 동작하지만, `adduser`에서는 전체 `--uid` 플래그가 필요하다.

## Q3. 사용자 비밀번호 잠금

**문제:** `han01` 계정의 비밀번호를 잠그는 명령어는?

| 선택지 | 명령어 |
|--------|--------|
| A | `passwd -d han01` |
| B | `passwd -u han01` |
| C | `passwd -l han01` |
| D | `passwd -x han01` |

**정답:** C. `passwd -l han01`

**해설:** `passwd`의 `-l`(lock) 옵션은 `/etc/shadow`의 암호화된 비밀번호 앞에 `!`를 추가하여 사용자의 비밀번호를 비활성화하고, 인증을 불가능하게 만든다. `-d`는 비밀번호를 삭제하고, `-u`는 잠긴 비밀번호를 해제하며, `-x`는 최대 비밀번호 사용 기간을 설정한다.

## Q4. 디스크 쿼터 할당

**문제:** 사용자 계정 `han01`에 디스크 쿼터를 할당하는 명령어는?

| 선택지 | 명령어 |
|--------|--------|
| A | `quotaon han01` |
| B | `edquota -u han01` |
| C | `edquota han01` |
| D | `quotacheck -u han01` |

**정답:** B. `edquota -u han01`

**해설:** `edquota` 명령어는 사용자의 디스크 쿼터 제한을 설정하기 위한 편집기를 연다. `-u` 옵션은 쿼터가 사용자용임을 명시적으로 지정한다(그룹용은 `-g`). `quotaon`은 쿼터 적용을 활성화하고, `quotacheck`는 파일 시스템의 디스크 사용량을 스캔하며, `-u` 없는 `edquota`는 모호할 수 있다.

## Q5. 로그인 셸 지정

**문제:** Bourne 셸을 로그인 셸로 하여 사용자 계정 `han01`을 생성하는 명령어는?

| 선택지 | 명령어 |
|--------|--------|
| A | `useradd -b sh han01` |
| B | `useradd -s sh han01` |
| C | `useradd -b /bin/sh han01` |
| D | `useradd -s /bin/sh han01` |

**정답:** D. `useradd -s /bin/sh han01`

**해설:** `useradd`의 `-s` 옵션은 새 사용자의 로그인 셸을 설정한다. 셸은 이름만이 아니라 전체 절대 경로(`/bin/sh`)로 지정해야 한다. `-b` 옵션은 셸이 아닌 홈 디렉터리의 기본 디렉터리를 설정한다. 따라서 `-s /bin/sh`가 올바른 조합이다.

## Q6. 최대 비밀번호 사용 기간 설정

**문제:** `han01`의 최대 비밀번호 사용 기간을 200일로 설정하는 명령어는?

| 선택지 | 명령어 |
|--------|--------|
| A | `useradd -x 200 han01` |
| B | `usermod -x 200 han01` |
| C | `chage -M 200 han01` |
| D | `chage -m 200 han01` |

**정답:** C. `chage -M 200 han01`

**해설:** `chage` 명령어는 비밀번호 만료 정책을 관리한다. `-M`(대문자) 옵션은 비밀번호가 유효한 최대 일수를 설정한다. `-m`(소문자) 옵션은 비밀번호 변경 간 최소 일수를 설정한다. 표준 구현에서 `useradd`와 `usermod`는 이 목적으로 `-x`를 사용하지 않는다.

## Q7. EUID 확인

**문제:** EUID(Effective User ID)를 확인하는 데 사용할 수 있는 명령어는?

**정답:** `whoami`, `id`

**해설:** `whoami` 명령어는 현재 EUID에 연관된 유효 사용자 이름을 출력한다. `id` 명령어는 실제(real) 및 유효(effective) 사용자 ID와 그룹 ID를 표시한다. 두 명령어 모두 유효 사용자 신원을 확인하는 데 사용할 수 있으며, 특히 `su`나 `sudo`를 사용하여 사용자 컨텍스트를 전환할 때 유용하다.

## Q8. 비밀번호 저장 파일

**문제:** 사용자 비밀번호가 저장되는 파일은?

**정답:** `/etc/shadow`

**해설:** 사용자 비밀번호는 해시된 형태로 `/etc/shadow`에 저장된다. 이 파일은 root만 읽을 수 있어, `/etc/passwd`에 비밀번호 해시를 저장하던 이전 방식보다 보안성이 향상되었다. `/etc/shadow`의 각 줄에는 사용자 이름, 해시된 비밀번호, 그리고 다양한 비밀번호 만료 관련 필드가 포함된다.

## Q9. 기본 그룹 정보

**문제:** 사용자의 기본(주) 그룹을 확인할 수 있는 파일은?

**정답:** `/etc/passwd`

**해설:** `/etc/passwd` 파일에는 콜론으로 구분된 7개 필드로 기본 사용자 계정 정보가 포함된다: `username:password:UID:GID:GECOS:home:shell`. 네 번째 필드(GID)가 사용자의 주 그룹 ID를 지정한다. 이 GID에 해당하는 그룹 이름은 `/etc/group`에서 조회할 수 있다.

## Q10. 사용자 삭제 시 `-r` 옵션

**문제:** 사용자 계정 삭제 시 `-r` 옵션의 의미는?

**정답:** 사용자 계정과 함께 사용자의 홈 디렉터리(및 그 내용)를 삭제한다.

**해설:** `userdel -r username`을 사용하면, `-r` 옵션은 계정 항목과 함께 사용자의 홈 디렉터리와 메일 스풀이 제거되도록 보장한다. `-r` 없이 사용하면, `/etc/passwd`와 `/etc/shadow`에서 사용자 계정만 삭제되며, 홈 디렉터리와 파일은 시스템에 남아있게 된다.
