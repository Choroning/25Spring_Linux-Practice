## Q1. 파일 유형 식별

**문제:** 다음 `ls -l` 출력에서 `mylink`는 어떤 유형의 파일인가?

```
lrwxrwxrwx 1 user1 user1 15 Mar 21 10:00 mylink -> /home/user1/data
```

| 선택지 | 파일 유형 |
|--------|-----------|
| A | 일반 파일 |
| B | 디렉터리 |
| C | 심볼릭 링크 |
| D | 블록 장치 |

**정답:** C. 심볼릭 링크

**해설:** `ls -l` 출력의 첫 번째 문자가 파일 유형을 나타낸다. `l`은 심볼릭 링크, `-`는 일반 파일, `d`는 디렉터리, `b`는 블록 장치, `c`는 문자 장치, `p`는 파이프, `s`는 소켓을 의미한다. `->` 화살표도 심볼릭 링크가 어디를 가리키는지를 나타낸다.

## Q2. 디렉터리 계층 구조

**문제:** `hostname`, `passwd`, `fstab`과 같은 시스템 전체 설정 파일이 저장되는 디렉터리는?

| 선택지 | 디렉터리 |
|--------|----------|
| A | `/home` |
| B | `/var` |
| C | `/etc` |
| D | `/usr` |

**정답:** C. `/etc`

**해설:** `/etc` 디렉터리에는 시스템 전체 설정 파일이 포함되어 있다. 예를 들어, `/etc/hostname`은 시스템 호스트 이름, `/etc/passwd`는 사용자 계정 정보, `/etc/fstab`은 파일시스템 마운트 항목을 저장한다. `/home`은 사용자 디렉터리, `/var`는 로그와 같은 가변 데이터, `/usr`은 사용자 시스템 자원을 저장한다.

## Q3. 절대 경로 vs. 상대 경로

**문제:** 현재 작업 디렉터리가 `/home/user1/Documents`일 때, 상대 경로를 사용하여 `/home/user2/file.txt`를 올바르게 참조하는 것은?

| 선택지 | 경로 |
|--------|------|
| A | `../user2/file.txt` |
| B | `./user2/file.txt` |
| C | `../../user2/file.txt` |
| D | `~/user2/file.txt` |

**정답:** C. `../../user2/file.txt`

**해설:** `/home/user1/Documents`에서 시작하여, 첫 번째 `..`으로 `/home/user1`로, 두 번째 `..`으로 `/home`으로 올라간 후, `user2/file.txt`로 내려가면 `/home/user2/file.txt`가 된다. 선택지 A는 `/home/user1/user2/file.txt`로 해석된다. 선택지 B는 현재 디렉터리를 찾는다. 선택지 D는 현재 사용자의 홈으로 확장되는 `~`를 사용하므로 다른 사용자의 상대 경로가 아니다.

## Q4. 중첩 디렉터리 생성

**문제:** `projects/2025/spring/` 디렉터리 구조를 하나의 명령어로 생성해야 하지만, 상위 디렉터리가 하나도 존재하지 않는다. 어떤 명령어가 성공하는가?

| 선택지 | 명령어 |
|--------|--------|
| A | `mkdir projects/2025/spring` |
| B | `mkdir -p projects/2025/spring` |
| C | `mkdir -r projects/2025/spring` |
| D | `rmdir -p projects/2025/spring` |

**정답:** B. `mkdir -p projects/2025/spring`

**해설:** `-p` (parents) 옵션은 경로를 따라 필요한 모든 상위 디렉터리를 생성한다. `-p` 없이는 중간 디렉터리가 존재하지 않으면 `mkdir`이 실패한다. 선택지 C는 `mkdir`에 존재하지 않는 `-r` 플래그를 사용한다. 선택지 D는 디렉터리를 생성하는 것이 아니라 제거한다.

## Q5. 파일 내용 보기

**문제:** 새 항목이 추가되는 로그 파일을 실시간으로 모니터링하고 싶다. 가장 적절한 명령어는?

| 선택지 | 명령어 |
|--------|--------|
| A | `cat /var/log/syslog` |
| B | `head -f /var/log/syslog` |
| C | `tail -f /var/log/syslog` |
| D | `less -f /var/log/syslog` |

**정답:** C. `tail -f /var/log/syslog`

**해설:** `tail -f` (follow)는 파일의 마지막 10줄을 표시한 후, 새로운 줄이 추가될 때마다 계속 감시하고 표시한다. 이는 실시간 로그 모니터링에 매우 유용하다. `cat`은 전체 파일을 한꺼번에 출력한다. `head`에는 `-f` 옵션이 없다. `less`는 파일을 표시할 수 있지만 같은 방식으로 새로운 출력을 지속적으로 추적하지는 않는다.

## Q6. 숨김 파일

**문제:** `.`으로 시작하는 숨김 파일을 포함하여 현재 디렉터리의 모든 파일을 나열하려면?

**정답:** `ls -a`

**해설:** `-a` (all) 옵션을 사용하면 `ls`가 이름이 `.`으로 시작하는 숨김 파일과 디렉터리를 포함하여 표시한다. `-a` 없이는 `.bashrc`, `.profile`, `.ssh/`와 같은 파일이 표시되지 않는다. `ls -la`처럼 다른 옵션과 결합하여 숨김 파일을 상세 형식으로 볼 수도 있다.

## Q7. /root와 /의 구분

**문제:** Linux 디렉터리 계층 구조에서 `/`와 `/root`의 차이를 설명하시오.

**정답:** `/`는 루트 디렉터리로, 모든 다른 디렉터리가 분기되는 전체 파일시스템 계층 구조의 최상위 디렉터리이다. `/root`는 root(슈퍼유저) 계정의 홈 디렉터리로, 보안 및 가용성의 이유로 `/home/root`가 아닌 `/root`에 위치한다.

**해설:** 일반 사용자는 `/home/username` 아래에 홈 디렉터리가 있지만, root 사용자의 홈은 `/root`이다. 이러한 분리는 `/home` 파일시스템이 마운트되지 않더라도 root가 여전히 로그인하고 운영할 수 있도록 보장한다. "root"라는 단어를 모두 포함하고 있지만 완전히 다른 디렉터리이다.

## Q8. ls -l 출력 해석

**문제:** 다음 출력에서 `report.txt` 파일의 하드 링크 수와 파일 크기는?

```
-rw-r--r-- 3 user1 staff 2048 Mar 21 14:00 report.txt
```

**정답:** 이 파일은 3개의 하드 링크를 가지며, 크기는 2048바이트이다.

**해설:** `ls -l` 출력에서 권한 문자열 바로 다음의 숫자가 하드 링크 수(이 경우 3)이다. 날짜 앞의 숫자가 파일 크기(바이트 단위, 2048)이다. 하드 링크 수 3은 동일한 inode를 가리키는 3개의 디렉터리 항목(파일 이름)이 있음을 의미한다.
