## 1. Linux 파일과 디렉터리

### 1.1 모든 것은 파일이다

Linux에서는 장치와 디렉터리를 포함하여 거의 모든 것이 파일로 취급된다.

### 1.2 파일 유형

| 유형 | `ls -l`에서의 기호 | 설명 |
|------|-------------------|------|
| 일반 파일 | `-` | 텍스트 파일, 실행 파일, 이미지, 데이터 파일 |
| 디렉터리 | `d` | 다른 파일 및 디렉터리에 대한 참조를 포함 |
| 심볼릭 링크 | `l` | 다른 파일을 가리키는 포인터 (Windows 바로가기와 유사) |
| 블록 장치 | `b` | 블록 지향 장치 (예: 하드 디스크 `/dev/sda`) |
| 문자 장치 | `c` | 문자 지향 장치 (예: 키보드, 터미널) |
| 파이프 | `p` | 프로세스 간 통신을 위한 이름 있는 파이프 |
| 소켓 | `s` | 네트워크 통신 엔드포인트 |

### 1.3 파일 유형 식별

```bash
# 'file' 명령어를 사용하여 파일 유형 확인
$ file .profile
.profile: ASCII text

$ file /bin/bash
/bin/bash: ELF 64-bit LSB pie executable, x86-64

$ file /home
/home: directory
```

## 2. 디렉터리 계층 구조

### 2.1 디렉터리 트리

```
/                          # 루트 디렉터리 (계층 구조의 최상위)
├── bin/                   # 필수 사용자 명령어 바이너리 (-> /usr/bin)
├── boot/                  # 부트 로더 파일 (커널, initramfs)
├── dev/                   # 장치 파일
├── etc/                   # 시스템 설정 파일
├── home/                  # 사용자 홈 디렉터리
│   ├── user1/
│   └── user2/
├── lib/                   # 필수 공유 라이브러리
├── media/                 # 이동식 미디어 마운트 지점
├── mnt/                   # 임시 마운트 지점
├── opt/                   # 선택적/추가 패키지
├── proc/                  # 프로세스/커널 정보용 가상 파일시스템
├── root/                  # Root 사용자의 홈 디렉터리
├── run/                   # 실행 중인 서비스의 런타임 데이터
├── srv/                   # 서비스 데이터 (FTP, 웹)
├── sys/                   # 커널 및 장치 정보
├── tmp/                   # 임시 파일 (재부팅 시 삭제됨)
├── usr/                   # 사용자 시스템 자원
│   ├── bin/               # 사용자 명령어
│   ├── lib/               # 라이브러리
│   └── local/             # 로컬 설치 소프트웨어
└── var/                   # 가변 데이터 (로그, 스풀, 캐시)
    └── log/               # 시스템 로그 파일
```

### 2.2 주요 디렉터리

| 디렉터리 | 용도 | 비고 |
|----------|------|------|
| `/` | 파일시스템의 루트 | 모든 것이 여기서 시작됨 |
| `/home` | 사용자 홈 디렉터리 | 각 사용자는 `/home/username`을 가짐 |
| `/etc` | 설정 파일 | 시스템 전체 설정 |
| `/var` | 가변 데이터 | 로그, 메일, 데이터베이스 |
| `/tmp` | 임시 파일 | 재부팅 시 삭제됨 |
| `/proc` | 프로세스 정보 | 가상 파일시스템 (메모리에만 존재) |
| `/usr` | Unix 시스템 자원 | 실행 파일, 라이브러리, 문서 |
| `/root` | Root 사용자 홈 | `/`(루트 디렉터리)와 다름 |

> **핵심 포인트:** `/proc`는 하드 디스크가 아닌 메모리에만 존재한다. 실시간 커널 및 프로세스 정보를 제공한다.

## 3. 절대 경로와 상대 경로

### 3.1 절대 경로

- **항상 `/`**(루트 디렉터리)로 시작
- 루트에서 대상까지의 완전한 경로를 지정
- 현재 디렉터리와 관계없이 항상 같은 위치를 참조

```bash
/home/user1/Documents/report.txt
/etc/hostname
/usr/bin/ls
```

### 3.2 상대 경로

- **`/`로 시작하지 않음**
- 현재 작업 디렉터리를 기준으로 함
- `.`(현재 디렉터리)과 `..`(상위 디렉터리)를 사용

```bash
# 현재 디렉터리가 /home/user1인 경우
Documents/report.txt        # /home/user1/Documents/report.txt와 동일
../user2/file.txt           # /home/user2/file.txt와 동일
./script.sh                 # /home/user1/script.sh와 동일
```

### 3.3 특수 디렉터리 기호

| 기호 | 의미 | 예시 |
|------|------|------|
| `.` | 현재 디렉터리 | `./script.sh` |
| `..` | 상위 디렉터리 | `cd ..` |
| `~` | 홈 디렉터리 | `cd ~` 또는 `cd` |
| `~user` | 다른 사용자의 홈 | `cd ~user2` |
| `-` | 이전 디렉터리 | `cd -` |

## 4. 파일 및 디렉터리 이름 규칙

### 4.1 규칙

- 이름에 `/`를 사용할 수 없음 (경로 구분자임)
- 문자, 숫자, 하이픈(`-`), 밑줄(`_`), 마침표(`.`)를 사용할 수 있음
- 특수 문자 사용 자제: `> | : & space`
- **대소문자 구분:** `File.txt`와 `file.txt`는 서로 다른 파일
- `.`으로 시작하는 이름은 숨김 파일
- 널 문자(`\0`)는 사용할 수 없음
- 최대 파일 이름 길이: 일반적으로 255자

### 4.2 예시

| 좋은 이름 | 나쁜 이름 | 잘못된 이름 |
|-----------|-----------|-------------|
| `game.txt` | `&game` | `myhome/` |
| `hello.c` | `*dir` | `/test` |
| `test` | `my home` | `bad/name` |
| `sample11` | `game\` | |

## 5. 디렉터리 명령어

### 5.1 pwd — 현재 작업 디렉터리 출력

```bash
$ pwd
/home/user1
```

### 5.2 cd — 디렉터리 변경

```bash
cd /tmp                 # 절대 경로
cd ../usr/lib           # 상대 경로
cd ~                    # 홈 디렉터리
cd                      # 홈 디렉터리 (단축키)
cd -                    # 이전 디렉터리
cd ..                   # 상위 디렉터리
```

### 5.3 ls — 디렉터리 내용 목록

```bash
ls                      # 현재 디렉터리 목록
ls -a                   # 숨김 파일 표시 (.으로 시작하는 파일)
ls -l                   # 상세 형식 (자세한 정보)
ls -la                  # 상세 형식 + 숨김 파일
ls -lh                  # 사람이 읽기 쉬운 파일 크기
ls -R                   # 재귀적 목록
ls -F                   # 유형 표시자 추가 (*=실행 파일, /=디렉터리, @=링크)
ls -i                   # inode 번호 표시
ls -d                   # 내용이 아닌 디렉터리 자체 표시
ls -t                   # 수정 시간순 정렬
ls -S                   # 파일 크기순 정렬
```

**`ls -l` 출력 이해:**

```
drwxr-xr-x  2  user1  user1  4096  Mar 21 12:05  Documents
│└─┬──┘└┬─┘ │   │       │      │       │            │
│  │    │   │   │       │      │       │            └── 파일 이름
│  │    │   │   │       │      │       └── 최종 수정일
│  │    │   │   │       │      └── 크기 (바이트)
│  │    │   │   │       └── 그룹 소유자
│  │    │   │   └── 파일 소유자
│  │    │   └── 하드 링크 수
│  │    └── 기타 사용자 권한 (r-x)
│  └── 그룹 권한 (r-x)
│       소유자 권한 (rwx)
└── 파일 유형 (d=디렉터리, -=파일, l=링크)
```

### 5.4 mkdir — 디렉터리 생성

```bash
mkdir mydir                     # 단일 디렉터리 생성
mkdir -p parent/child/grandchild  # 중첩 디렉터리 생성
mkdir dir1 dir2 dir3            # 여러 디렉터리 생성
```

### 5.5 rmdir — 빈 디렉터리 제거

```bash
rmdir mydir                     # 빈 디렉터리만 제거
rmdir -p parent/child/grandchild  # 중첩된 빈 디렉터리 제거
```

### 5.6 유사 명령어: dir, vdir

```bash
dir                 # ls와 유사 (기본 색상 없음)
vdir                # ls -l과 유사
```

## 6. 파일 내용 명령어

### 6.1 cat — 연결 및 표시

```bash
cat file.txt                    # 전체 파일 표시
cat -n file.txt                 # 줄 번호 표시
cat file1.txt file2.txt         # 여러 파일 연결
cat > newfile.txt               # stdin에서 파일 생성 (Ctrl+D로 종료)
cat >> existingfile.txt         # stdin에서 파일에 추가
```

### 6.2 more와 less — 파일 페이지 넘기기

```bash
more file.txt                   # 앞으로만 페이지 넘기기
less file.txt                   # 앞뒤로 페이지 넘기기 (권장)
```

**less 탐색:**

| 키 | 동작 |
|----|------|
| `Space` / `f` | 다음 페이지 |
| `b` | 이전 페이지 |
| `g` | 처음으로 이동 |
| `G` | 끝으로 이동 |
| `/pattern` | 앞으로 검색 |
| `?pattern` | 뒤로 검색 |
| `n` | 다음 검색 결과 |
| `q` | 종료 |

### 6.3 head와 tail — 파일 부분 보기

```bash
head file.txt                   # 처음 10줄 (기본값)
head -n 20 file.txt             # 처음 20줄
head -n -5 file.txt             # 마지막 5줄을 제외한 모든 줄

tail file.txt                   # 마지막 10줄 (기본값)
tail -n 20 file.txt             # 마지막 20줄
tail -f /var/log/syslog         # 실시간으로 로그 추적
```

> **핵심 포인트:** `tail -f`는 로그 파일을 실시간으로 모니터링할 때 매우 유용하다.

## 7. 모범 사례

### 7.1 탐색 팁

- `Tab` 자동 완성을 사용하여 경로의 오타 방지
- `cd -`를 사용하여 두 디렉터리 간 전환
- `pushd`와 `popd`를 사용한 디렉터리 스택 관리
- 스크립트에서는 신뢰성을 위해 절대 경로 사용 권장

### 7.2 흔한 실수

| 실수 | 올바른 방법 |
|------|------------|
| 파일 이름에 공백 사용 | 밑줄이나 하이픈 사용 |
| 대소문자 구분 잊기 | Linux는 대소문자를 구분함 |
| `/root`와 `/` 혼동 | `/root`는 root의 홈이고, `/`는 파일시스템 루트 |
| `rm -rf /` 사용 | 절대 실행하지 말 것 — 모든 것이 삭제됨 |
