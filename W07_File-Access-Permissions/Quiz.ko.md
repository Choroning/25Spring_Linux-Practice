## Q1. 권한 문자열 해석

**문제:** 권한 문자열 `drwxr-x---`는 무엇을 나타내는가?

| 선택지 | 설명 |
|--------|-------------|
| A | 모든 사용자에게 전체 접근이 가능한 일반 파일 |
| B | 소유자에게 전체 접근 권한이 있고, 그룹은 읽기 및 진입이 가능하며, 기타 사용자는 접근 불가인 디렉터리 |
| C | 모든 사용자가 읽기와 실행이 가능한 디렉터리 |
| D | 제한된 권한을 가진 심볼릭 링크 |

**정답:** B. 소유자에게 전체 접근 권한이 있고, 그룹은 읽기 및 진입이 가능하며, 기타 사용자는 접근 불가인 디렉터리

**해설:** 첫 번째 문자 `d`는 디렉터리를 나타낸다. 소유자 권한 `rwx`(읽기, 쓰기, 실행)는 내용 나열, 파일 생성/삭제, 디렉터리 진입을 포함한 전체 접근을 허용한다. 그룹 권한 `r-x`는 내용 나열과 진입은 가능하지만 파일 생성이나 삭제는 불가하다. 기타 `---`는 어떤 접근도 불가하다.

## Q2. 8진수 권한 변환

**문제:** 권한 `rw-r-xr--`의 8진수 표현은?

| 선택지 | 8진수 |
|--------|-------|
| A | `654` |
| B | `645` |
| C | `754` |
| D | `544` |

**정답:** A. `654`

**해설:** 각 권한 세트는 r=4, w=2, x=1을 더하여 8진수로 변환한다. 소유자 `rw-` = 4+2+0 = 6. 그룹 `r-x` = 4+0+1 = 5. 기타 `r--` = 4+0+0 = 4. 합산: `654`.

## Q3. 기호 모드로 chmod 사용

**문제:** `script.sh`에 대해 소유자에게 실행 권한을 추가하고 기타 사용자의 쓰기 권한을 제거하는 명령어는?

| 선택지 | 명령어 |
|--------|---------|
| A | `chmod u+x,o+w script.sh` |
| B | `chmod u+x,o-w script.sh` |
| C | `chmod a+x,o-w script.sh` |
| D | `chmod u-x,o+w script.sh` |

**정답:** B. `chmod u+x,o-w script.sh`

**해설:** `u+x`는 소유자(user)에게 실행 권한을 추가하고, `o-w`는 기타 사용자에서 쓰기 권한을 제거한다. 여러 권한 변경은 쉼표로 결합할 수 있다. 선택지 A는 기타에게 쓰기를 제거하는 대신 추가한다. 선택지 C는 소유자뿐만 아니라 모든 사용자에게 실행 권한을 추가한다. 선택지 D는 두 작업을 모두 반대로 한다.

## Q4. umask 이해

**문제:** umask가 `027`로 설정된 경우, 새로 생성된 일반 파일의 권한은?

| 선택지 | 권한 |
|--------|-------------|
| A | `750` (`rwxr-x---`) |
| B | `640` (`rw-r-----`) |
| C | `027` (`----w-rwx`) |
| D | `666` (`rw-rw-rw-`) |

**정답:** B. `640` (`rw-r-----`)

**해설:** 새 파일의 기본 권한은 `666`이다(파일은 기본적으로 실행 권한을 받지 않음). umask를 빼면: `666 - 027 = 640`. 소유자는 `rw-` (6), 그룹은 `r--` (4), 기타는 `---` (0). 디렉터리의 경우 기본값이 `777`이므로 새 디렉터리는 `750`이 된다.

## Q5. 소유권 변경

**문제:** `data/`와 그 모든 내용의 소유자를 `admin`으로, 그룹을 `staff`로 변경하는 명령어는?

| 선택지 | 명령어 |
|--------|---------|
| A | `sudo chown admin:staff data/` |
| B | `sudo chown -R admin:staff data/` |
| C | `sudo chgrp -R admin:staff data/` |
| D | `sudo chmod -R admin:staff data/` |

**정답:** B. `sudo chown -R admin:staff data/`

**해설:** `chown`은 소유권을 변경하고, `user:group` 구문은 소유자와 그룹을 동시에 설정하며, `-R`은 모든 파일과 하위 디렉터리에 재귀적으로 적용한다. 선택지 A는 `-R`이 없어 디렉터리 자체에만 적용되고 내용에는 적용되지 않는다. `chgrp`는 그룹만 변경한다. `chmod`는 소유권이 아닌 권한을 변경한다.

## Q6. 특수 권한 — Sticky Bit

**문제:** `/tmp` 디렉터리의 권한이 다음과 같다:

```
drwxrwxrwt 15 root root 4096 Mar 21 10:00 /tmp
```

권한 문자열 끝의 `t`는 무엇을 의미하는가?

**정답:** `t`는 sticky bit가 설정되어 있음을 나타낸다. 디렉터리에 sticky bit가 설정되면, 다른 사용자에게 쓰기 권한이 있더라도 파일 소유자(또는 root)만 해당 디렉터리 내의 파일을 삭제하거나 이름을 변경할 수 있다.

**해설:** Sticky bit가 없으면 디렉터리에 쓰기 권한이 있는 누구나 그 안의 모든 파일을 삭제할 수 있다. `/tmp`의 sticky bit는 `/tmp`가 모든 사용자에게 쓰기 가능(`rwxrwxrwx`)하기 때문에 필수적이며, 이것이 없으면 다른 사용자의 임시 파일을 아무나 삭제할 수 있다. `t`는 기타의 실행 위치에 나타나며, 소문자 `t`는 실행 권한도 설정되어 있음을 의미하고, 대문자 `T`는 sticky는 설정되었지만 기타의 실행 권한은 없음을 의미한다.

## Q7. SetUID 권한

**문제:** `/usr/bin/passwd` 명령어에 SUID 비트가 설정되어 있는 이유와, 이를 제거하면 어떻게 되는가?

**정답:** SUID 비트는 어떤 사용자가 실행하든 `passwd` 명령어가 root 권한으로 실행되도록 한다. 이는 비밀번호 변경 시 root만 쓸 수 있는 `/etc/shadow`에 쓰기가 필요하기 때문이다. SUID 비트가 제거되면 일반 사용자는 비밀번호를 변경하려 할 때 "Permission denied" 오류가 발생한다.

**해설:** 실행 파일에 SUID(`s`, 소유자 실행 위치)가 설정되면, 호출자의 권한 대신 파일 소유자의 권한으로 실행된다. `/usr/bin/passwd`는 root 소유이므로, 이를 실행하는 모든 사용자는 `/etc/shadow`를 수정할 수 있는 일시적인 root 수준 접근 권한을 얻게 된다. SUID 비트는 `ls -l` 출력에서 `s`로 표시된다: `-rwsr-xr-x`.

## Q8. 누구나 쓸 수 있는 파일 찾기

**문제:** "Permission denied" 오류를 억제하면서 시스템의 모든 누구나 쓸 수 있는 일반 파일을 찾는 명령어를 작성하라.

**정답:** `find / -type f -perm -o+w 2>/dev/null`

**해설:** `-type f`는 일반 파일로 검색을 제한하고, `-perm -o+w`는 기타 사용자가 쓰기 권한을 가진 파일과 일치한다. `2>/dev/null`은 stderr(제한된 디렉터리에서 발생하는 "Permission denied" 메시지)를 `/dev/null`로 리다이렉트하여 출력을 깔끔하게 유지한다. 이는 시스템의 모든 사용자가 수정할 수 있는 파일을 식별하기 위한 일반적인 보안 감사 명령어이다.
