## 1. 파일 권한 이해

### 1.1 사용자 범주

모든 파일에는 세 가지 사용자 범주가 존재한다:

| 범주 | 기호 | 설명 |
|----------|--------|-------------|
| **소유자 (User)** | `u` | 파일을 생성한 사용자 |
| **그룹 (Group)** | `g` | 파일의 그룹에 속한 사용자들 |
| **기타 (Others)** | `o` | 시스템의 모든 다른 사용자 |
| **전체 (All)** | `a` | 세 범주 모두 |

### 1.2 권한 유형

| 권한 | 기호 | 파일에 대한 의미 | 디렉터리에 대한 의미 |
|-----------|--------|-----------|-----------------|
| **읽기** | `r` (4) | 파일 내용 보기 | 디렉터리 내용 나열 (`ls`) |
| **쓰기** | `w` (2) | 파일 내용 수정 | 디렉터리 내에 파일 생성/삭제 |
| **실행** | `x` (1) | 프로그램으로 실행 | 디렉터리 진입 (`cd`) |

```bash
$ ls -l myfile.txt
-rw-r--r-- 1 user1 group1 1024 Mar 21 10:00 myfile.txt
```

## 2. 권한 표현 방식

### 2.1 기호 표기법

```
-  rwx  r-x  r--
│  └┬┘  └┬┘  └┬┘
│   │    │    └── 기타: 읽기만 가능
│   │    └── 그룹: 읽기 + 실행
│   └── 소유자: 읽기 + 쓰기 + 실행
└── 파일 유형: 일반 파일
```

### 2.2 8진수(숫자) 표기법

| 권한 | 2진수 | 8진수 |
|-----------|--------|-------|
| `---` | 000 | 0 |
| `--x` | 001 | 1 |
| `-w-` | 010 | 2 |
| `-wx` | 011 | 3 |
| `r--` | 100 | 4 |
| `r-x` | 101 | 5 |
| `rw-` | 110 | 6 |
| `rwx` | 111 | 7 |

**자주 사용되는 권한 값:**

| 8진수 | 기호 | 일반적인 용도 |
|-------|----------|-------------|
| `755` | `rwxr-xr-x` | 실행 파일, 디렉터리 |
| `644` | `rw-r--r--` | 일반 파일 |
| `700` | `rwx------` | 개인 디렉터리/스크립트 |
| `600` | `rw-------` | 개인 파일 (예: SSH 키) |
| `777` | `rwxrwxrwx` | 전체 접근 (운영 환경에서 사용 금지) |
| `000` | `----------` | 접근 불가 |

## 3. chmod로 권한 변경

### 3.1 기호 모드

```bash
chmod u+x file.sh            # 소유자에게 실행 권한 추가
chmod g+rw file.txt           # 그룹에게 읽기+쓰기 권한 추가
chmod o-w file.txt            # 기타에서 쓰기 권한 제거
chmod a+r file.txt            # 전체에게 읽기 권한 추가
chmod u=rwx,g=rx,o=r file     # 정확한 권한 설정
chmod u+x,g-w file            # 복수 변경
chmod -R 755 directory/       # 재귀적 권한 변경
```

**연산자:**
- `+` : 권한 추가
- `-` : 권한 제거
- `=` : 정확한 권한 설정

### 3.2 8진수 모드

```bash
chmod 755 script.sh           # rwxr-xr-x
chmod 644 document.txt        # rw-r--r--
chmod 700 private/            # rwx------
chmod 600 id_rsa              # rw-------
chmod -R 755 /var/www/html/   # 재귀적
```

## 4. 소유권 변경

### 4.1 chown — 소유자 변경

```bash
sudo chown newuser file.txt               # 소유자 변경
sudo chown newuser:newgroup file.txt      # 소유자와 그룹 동시 변경
sudo chown :newgroup file.txt             # 그룹만 변경
sudo chown -R newuser:newgroup directory/ # 재귀적
```

### 4.2 chgrp — 그룹 변경

```bash
sudo chgrp newgroup file.txt              # 그룹 변경
sudo chgrp -R newgroup directory/         # 재귀적
```

### 4.3 소유권 확인

```bash
ls -l file.txt
# -rw-r--r-- 1 user1 group1 1024 Mar 21 10:00 file.txt
#               ^^^^^  ^^^^^^
#               소유자  그룹

# 소속 그룹 확인
groups
id
```

## 5. 기본 권한과 umask

### 5.1 umask 동작 원리

**umask**(사용자 파일 생성 마스크)는 새 파일과 디렉터리의 기본 권한을 결정한다.

```
기본 파일 권한:      666 (rw-rw-rw-)
기본 디렉터리 권한: 777 (rwxrwxrwx)

실제 권한 = 기본값 - umask
```

### 5.2 일반적인 umask 값

| umask | 파일 결과 | 디렉터리 결과 |
|-------|-------------|-----------------|
| `022` | `644` (rw-r--r--) | `755` (rwxr-xr-x) |
| `002` | `664` (rw-rw-r--) | `775` (rwxrwxr-x) |
| `077` | `600` (rw-------) | `700` (rwx------) |
| `000` | `666` (rw-rw-rw-) | `777` (rwxrwxrwx) |

```bash
# 현재 umask 확인
umask              # 8진수 형식
umask -S           # 기호 형식

# umask 설정
umask 022          # 대부분의 시스템에서 표준
umask 077          # 제한적 (개인 파일)

# 영구 설정 (~/.bashrc에 추가)
echo "umask 022" >> ~/.bashrc
```

## 6. 특수 권한

### 6.1 SetUID (SUID)

실행 파일에 설정하면, 프로그램이 호출자의 권한이 아닌 **소유자의 권한** 으로 실행된다.

```bash
# SUID 설정
chmod u+s program           # 기호 방식
chmod 4755 program          # 8진수 방식 (4 접두사)

# 예시: passwd 명령어
$ ls -l /usr/bin/passwd
-rwsr-xr-x 1 root root 68208 Mar 14 2022 /usr/bin/passwd
#   ^
#   s = SUID가 설정됨 (root 권한으로 실행)
```

### 6.2 SetGID (SGID)

- **파일:** 프로그램이 그룹의 권한으로 실행
- **디렉터리:** 새 파일이 디렉터리의 그룹을 상속

```bash
# SGID 설정
chmod g+s directory/        # 기호 방식
chmod 2755 directory/       # 8진수 방식 (2 접두사)
```

### 6.3 Sticky Bit

디렉터리에 설정하면, 파일 소유자(또는 root)만 해당 디렉터리 내의 파일을 삭제할 수 있다.

```bash
# Sticky bit 설정
chmod +t directory/         # 기호 방식
chmod 1755 directory/       # 8진수 방식 (1 접두사)

# 예시: /tmp 디렉터리
$ ls -ld /tmp
drwxrwxrwt 15 root root 4096 Mar 21 10:00 /tmp
#         ^
#         t = sticky bit (소유자만 자신의 파일을 삭제 가능)
```

### 6.4 특수 권한 요약

| 권한 | 8진수 | 파일에 대한 효과 | 디렉터리에 대한 효과 |
|-----------|-------|-----------------|----------------------|
| SUID | 4000 | 파일 소유자 권한으로 실행 | (효과 없음) |
| SGID | 2000 | 파일 그룹 권한으로 실행 | 새 파일이 그룹 상속 |
| Sticky | 1000 | (효과 없음) | 소유자만 삭제 가능 |

## 7. 모범 사례

### 7.1 보안 지침

- 운영 환경에서 `chmod 777` 사용 금지
- 민감한 파일은 `600`으로 유지 (예: SSH 개인 키, 비밀번호가 포함된 설정 파일)
- 기본적으로 디렉터리는 `755`, 일반 파일은 `644` 사용
- SUID/SGID 프로그램 최소화 (잠재적 보안 위험)
- 광범위한 권한 부여 대신 그룹을 활용하여 접근 공유

### 7.2 일반적인 보안 점검

```bash
# 시스템의 모든 SUID 파일 찾기
find / -perm -4000 -type f 2>/dev/null

# 모든 SGID 파일 찾기
find / -perm -2000 -type f 2>/dev/null

# 누구나 쓸 수 있는 파일 찾기
find / -perm -o+w -type f 2>/dev/null

# 소유자가 없는 파일 찾기
find / -nouser 2>/dev/null
```
