## Q1. 셸의 역할

**문제:** 다음 중 Linux에서 셸의 역할을 가장 잘 설명한 것은?

| 선택지 | 설명 |
|--------|-------------|
| A | 하드웨어 자원을 직접 관리하는 프로그램 |
| B | 사용자와 커널 사이의 명령줄 해석기 |
| C | Linux 커널 자체 |
| D | 그래픽 데스크톱 환경 |

**정답:** B. 사용자와 커널 사이의 명령줄 해석기

**해설:** 셸은 사용자와 Linux 커널 사이의 중개자 역할을 한다. 사용자의 명령을 읽고 해석하여(변수, 와일드카드, 별칭 등을 확장) 커널에 전달하여 실행하게 한다. 셸은 또한 결과를 사용자에게 다시 표시한다. 셸은 커널 자체가 아니며, 하드웨어를 직접 관리하지도 않는다.

## Q2. 기본 셸 확인

**문제:** 현재 사용자의 기본 로그인 셸을 표시하는 명령어는?

| 선택지 | 명령어 |
|--------|---------|
| A | `echo $HOME` |
| B | `echo $SHELL` |
| C | `echo $PATH` |
| D | `echo $USER` |

**정답:** B. `echo $SHELL`

**해설:** `$SHELL` 환경 변수는 사용자의 기본 로그인 셸 경로를 저장한다(예: `/bin/bash`). `$HOME`은 홈 디렉터리, `$PATH`는 명령어 검색 경로, `$USER`는 사용자 이름을 표시한다. `$SHELL`은 기본 셸을 나타내며, 사용자가 수동으로 전환한 경우 현재 활성 셸과 다를 수 있다.

## Q3. 환경 변수와 지역 변수의 차이

**문제:** `MY_VAR="hello"`와 `export MY_VAR="hello"`의 차이점은?

| 선택지 | 설명 |
|--------|-------------|
| A | 차이가 없으며, 둘 다 환경 변수를 생성한다 |
| B | `export`는 변수를 자식 프로세스에서 사용 가능하게 하고, `export` 없이는 현재 셸에서만 유효하다 |
| C | `export`는 변수를 읽기 전용으로 만든다 |
| D | `export` 없이는 변수가 영구적이고, `export`를 사용하면 임시적이다 |

**정답:** B. `export`는 변수를 자식 프로세스에서 사용 가능하게 하고, `export` 없이는 현재 셸에서만 유효하다

**해설:** `export` 없이 설정된 변수는 현재 셸 세션에서만 보이는 지역(셸) 변수이다. `export`를 사용하면 환경 변수로 승격되어 자식 프로세스(스크립트, 서브셸, 셸에서 실행된 프로그램)에 상속된다. 예를 들어, 셸에서 실행한 스크립트는 export된 변수에 접근할 수 있지만 지역 변수에는 접근할 수 없다.

## Q4. 셸 초기화 파일

**문제:** 새로운 대화형 bash 셸 세션이 시작될 때마다(새 터미널 창 열기 포함) 실행되는 파일은?

| 선택지 | 파일 |
|--------|------|
| A | `~/.bash_profile` |
| B | `~/.bash_logout` |
| C | `~/.bashrc` |
| D | `/etc/hostname` |

**정답:** C. `~/.bashrc`

**해설:** `~/.bashrc`는 새로운 대화형 비로그인 셸이 시작될 때마다 읽히며, 데스크톱 환경에서 새 터미널 창을 여는 것도 포함된다. `~/.bash_profile`(또는 `~/.profile`)은 로그인 셸에서만 읽힌다(예: SSH 로그인, tty 로그인). `~/.bash_logout`은 로그인 셸이 종료될 때 실행된다. 이 때문에 별칭과 사용자 설정은 보통 `~/.bashrc`에 배치한다.

## Q5. 별칭 관리

**문제:** 현재 셸에서 `alias rm='rm -i'` 별칭을 만들었다. 재부팅 후 별칭이 사라졌다. 별칭을 영구적으로 만들려면 어떻게 해야 하는가?

**정답:** `~/.bashrc` 파일에 `alias rm='rm -i'` 줄을 추가한 후, `source ~/.bashrc`를 실행하거나 새 터미널을 열어 적용한다.

**해설:** 현재 셸에서 생성된 별칭은 세션이 종료되면 사라진다. 세션 간에 지속되려면 셸 초기화 파일에 정의해야 한다. `~/.bashrc`는 모든 새 대화형 셸에서 읽히기 때문에 표준 위치이다. `~/.bashrc`를 편집한 후 `source ~/.bashrc`를 실행하면 로그아웃 없이 변경사항이 적용된다.

## Q6. 인용 규칙

**문제:** `name="Linux"`가 설정된 상태에서, 각 명령어의 출력은?

```bash
echo 'Hello $name'
echo "Hello $name"
```

| 선택지 | 첫 번째 출력 / 두 번째 출력 |
|--------|-------------------------------------|
| A | `Hello Linux` / `Hello Linux` |
| B | `Hello $name` / `Hello $name` |
| C | `Hello $name` / `Hello Linux` |
| D | `Hello Linux` / `Hello $name` |

**정답:** C. `Hello $name` / `Hello Linux`

**해설:** 작은따옴표(`' '`)는 내부의 모든 문자를 문자 그대로 유지하므로 `$name`이 그대로 출력된다. 큰따옴표(`" "`)는 변수 확장을 허용하므로 `$name`이 값인 `Linux`로 치환된다. 이 구분은 변수 확장 시점을 제어해야 하는 셸 스크립트 작성 시 중요하다.

## Q7. 명령어 히스토리

**문제:** 키워드를 입력하여 명령어 히스토리를 대화형으로 검색할 수 있는 명령어 또는 단축키는?

| 선택지 | 방법 |
|--------|--------|
| A | `history \| grep keyword` |
| B | `Ctrl + R` |
| C | `!keyword` |
| D | `Ctrl + Z` |

**정답:** B. `Ctrl + R`

**해설:** `Ctrl + R`은 bash에서 역방향 증분 검색을 활성화하며, 문자를 입력하면 셸이 히스토리에서 가장 최근에 일치하는 명령을 동적으로 찾아준다. 선택지 A도 동작하지만 대화형이 아니며, 선택 또는 편집 없이 일치하는 행만 출력한다. `!keyword`는 미리보기 없이 "keyword"로 시작하는 마지막 명령을 즉시 실행한다. `Ctrl + Z`는 현재 포그라운드 프로세스를 일시 중지한다.

## Q8. PATH 수정

**문제:** `/opt/myapp/bin`에 사용자 지정 애플리케이션을 설치했으며, 어느 디렉터리에서나 해당 명령어를 사용하고 싶다. 이 디렉터리를 `PATH`에 추가하는 명령어를 작성하라.

**정답:** `export PATH=$PATH:/opt/myapp/bin`

**해설:** 이 명령은 기존 `PATH` 변수에 `/opt/myapp/bin`을 추가한다. 셸은 명령어를 찾을 때 `PATH`에 나열된 디렉터리를 콜론으로 구분하여 순서대로 검색한다. 뒤에 추가하면 기존 명령어에 대한 접근이 유지된다. 이를 영구적으로 만들려면 `~/.bashrc`에 해당 `export` 줄을 추가한다. 앞에 추가(`PATH=/opt/myapp/bin:$PATH`)하면 사용자 지정 디렉터리가 시스템 디렉터리보다 우선되어 보안 위험이 될 수 있다.
